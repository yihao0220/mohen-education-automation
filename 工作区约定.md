# 工作区约定

## 目的
- 这个仓库优先服务于修 bug 和迭代录入工具，不追求样本库和代码库完全混在一起。
- 后续新增文件时，优先判断它属于 `stable`、`sample`、`generated`、`local` 哪一类。

## 分类
### stable
- 正式代码和长期维护资产。
- 当前目录：
  - `墨痕快刀/`
  - `格式处理/`
  - `答案录入/`
  - `shared_core/`
  - `tools/`
  - `docs/`
  - `问题归档/`
  - `回归样本/` 中的清单、预检 JSON/MD 基线、P1a `DocumentFamilyReport.json/.md` 批次报告与说明文档
  - `README.md`
  - `AGENTS.md`
  - `requirements-poc.txt`

### sample
- 真实题目、真实答案、原始样本，主要用于复现问题。
- 当前典型目录：
  - `墨痕快刀/待录入文档/`
  - `格式处理/待清洗文件/`
  - `格式处理/原格式/`
- 这类文件默认不作为代码变更的一部分管理，除非明确要沉淀成固定回归样本。

### generated
- 程序跑出来的中间结果或输出结果。
- 当前典型文件：
  - `*_已清洗.docx`
  - `*_标准化.docx`
  - `*_审核清单.md`
  - `test_output.docx`
  - `转换后格式.docx`
- 这类文件默认不进版本控制，需要时现生现用。

### local
- 只和本机环境有关的目录。
- 当前典型目录：
  - `.venv/`
  - `.vscode/`
  - `.qoder/`
  - `.sisyphus/`
- `.qoder/repowiki/` 是本地生成的文档快照，可辅助检索，但不是项目事实的最终来源。

## 后续约定
- 修 bug 时，优先改 `stable`，只把 `sample` 当复现材料。
- 新的样本文档优先放进既有样本目录，不直接散落到仓库根目录。
- 新的临时输出不保留在根目录；确认无用后直接删除。
- 每次修复后，把现象、根因、测试样例写入 `问题归档/`。
- 如果 `.qoder/repowiki/` 与 `README.md`、`AGENTS.md`、`问题归档/INDEX.md` 冲突，以后者为准。
