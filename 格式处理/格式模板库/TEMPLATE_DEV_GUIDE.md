## 一、标准输出格式（必须遵守）

根据 `Standard output format.md`，清洗后的文档必须遵循以下格式：

```
1．B　
解析：白花豌豆提供花粉，作父本...
2．
B　解析：狗的长毛和短毛是一对相对性状...
3．D　
解析：豌豆为严格的自花授粉植物...
11．(1) 二　绿色子叶　Yy　(2) 2/3　...
解析：(1) 实验二中亲本黄色子叶自交...
```

### 格式要点

| 要素 | 规范 | 示例 |
|------|------|------|
| 题号 | 阿拉伯数字 + 全角点号 `．` | `1．`、`11．` |
| 选择题答案 | 大写字母（A-D），紧跟题号 | `1．B　` |
| 解析标记 | `解析：` + 内容 | `解析：白花豌豆...` |
| 大题答案 | 题号后换行，答案独立成行 | `11．\n(1) 二　绿色子叶...` |
| 空行 | 题与题之间不保留空行 | 连续输出 |

### 2.2 模板特征定义（TEMPLATE_FEATURES）

```python
TEMPLATE_FEATURES = {
    "name": "模板X_[中文名称]",
    "patterns": [
```

### 2.3 垃圾行过滤（GARBAGE_PATTERNS）

```python
GARBAGE_PATTERNS = [
    r"^课时分层作业",

    r"^一、选择题",
    r"^二、非选择题",

    r"^每小题",
```

### 2.4 正则模式定义

```python
# 题号识别（必须支持中文、英文、数字、书名号、括号开头）
```

### 2.5 字体格式设置（必须实现）

```python
def set_font_format(doc):
    """
    设置文档字体格式（所有模板统一）
    - 小四（12号）
    - 黑色
    - 宋体（中文）
    - Times New Roman（英文/数字）
    - 不加粗
    """
    try:
        for para in doc.Paragraphs:
            for run in para.Range.Words:
                run.Font.Size = 12
                run.Font.Color = 0  # 黑色
                run.Font.Name = "Times New Roman"
                run.Font.NameFarEast = "宋体"
                run.Font.Bold = False
        print("   ✓ 字体格式设置完成：小四、黑色、宋体/Times New Roman、不加粗")
    except Exception as e:
        print(f"   ! 字体设置失败: {e}")
```

### 2.7 匹配度计算（match_score）

```python
def match_score(doc):
    """
    计算文档与此模板的匹配度
    
    Args:
        doc: WPS 文档对象
    
    Returns:
        float: 匹配分数 (0-1)
    """
    total_lines = 0
    matched_lines = 0
    
    for p in doc.Paragraphs:
        text = p.Range.Text.strip()
        if not text:
            continue
        total_lines += 1
        for pattern in TEMPLATE_FEATURES["patterns"]:
            if re.search(pattern, text):
                matched_lines += 1
                break
    
    if total_lines == 0:
        return 0
    return matched_lines / total_lines
```

## 三、开发流程

### Step 1: 分析样本文档

1. 提供 `task_instruction.md` 描述问题
2. 提供样本 docx 文件放入 `待清洗文件/`
3. 分析文档结构：题号格式、答案格式、解析格式、垃圾行类型

### Step 2: 选择参考模板

| 模板 | 适用场景 | 可复制组件 |
|------|----------|-----------|
| template_a.py | 方括号格式 `[答案]` `[解析]` | 字体设置、质量校验、题号识别 |
| template_b.py | 中文标记格式 `【答案】` `【分析】` `【详解】` | 多标记解析、内循环收集 |

**原则：能复用就复用，但必须测试验证**

### Step 3: 编写模板

1. 复制参考模板作为基础
2. 修改 `TEMPLATE_FEATURES` 中的 `name` 和 `patterns`
3. 调整 `GARBAGE_PATTERNS` 匹配目标文档的垃圾行
4. 修改正则模式匹配目标文档的答案/解析格式
5. 调整 `clean_document` 中的解析逻辑

### Step 4: 注册模板

在 `__init__.py` 中添加新模板导入：

```python
from . import template_a
from . import template_b
from . import template_x  # 新模板

__all__ = ["template_a", "template_b", "template_x"]
```

### Step 5: 严格测试与修复闭环（关键！）

> ⚠️ **首次模板开发后必须进行严格测试，发现问题立即修复，记录经验，再次测试，直到完全符合标准格式。**

**测试检查清单**：

| 检查项 | 通过标准 | 常见问题 |
|--------|----------|----------|
| 题号连续性 | 1,2,3... 无跳号/重复 | 子标题被误识别为题号 |
| 选择题答案 | 题号后直接跟字母 | 答案丢失、格式错乱 |
| 解析完整性 | 包含所有续行（①②③④、故本题选X等） | 只收集首行，续行丢失 |
| 章节标题过滤 | 不混入题目内容 | 垃圾行过滤不全 |
| 半角/全角兼容 | 支持 `1．` 和 `1.` | 半角点号无法识别 |
| 大题小问 | (1)(2) 答案完整 | 小问续行丢失 |
| 字体格式 | 小四、宋体、黑色 | 字体设置未生效 |

**修复-记录-再测试流程**：

```
首次测试 → 发现问题 → 修复代码 → 记录问题到模板X开发经验记录.md
    ↑________________________________________________↓
                    再次测试（直到全部通过）
```

**禁止行为**：
- ❌ 首次开发后不测试直接交付
- ❌ 发现问题不记录直接修复
- ❌ 只测单文档不测边界情况（如：有无【解析】、半角全角混用）

### Step 6: 批量测试

1. 将多个样本放入 `待清洗文件/`
2. 运行 `格式处理/main.py`
3. 选择 `[1] 批量清洗待清洗文件目录`
4. 验证批量处理稳定性

## 四、常见陷阱与解决方案

### 4.1 题号识别错误

**问题**：章节编号 `3.7.2` 被识别为第3题  
**解决**：在 `GARBAGE_PATTERNS` 中添加 `r"^\d+\.\d+\."`

### 4.2 解析内容被截断

**问题**：【分析】内循环遇 `2．观察法` 误判为新题目  
**解决**：删除停止条件中的 `^\d+[．.]`，仅保留 `^【详解】|^【答案】`

### 4.3 大题小问答案丢失

**问题**：`(2)(3)(4)` 续行未被收集  
**解决**：使用内循环向后查找连续的小问行

### 4.4 字体格式不统一

**问题**：输出文档字体不一致  
**解决**：确保调用 `set_font_format(doc)` 且参数正确

### 4.5 模板未被加载

**问题**：新模板修改后不生效  
**解决**：检查 `__init__.py` 是否导出模板
