## 文档格式特征

### 来源文档
- **文件名**: 布心中学物理课时作业本（答案）.docx
- **学科**: 物理
- **学校**: 布心中学

### 格式特点

| 要素 | 格式 | 示例 |
|------|------|------|
| 选择题答案 | `题号. 答案`（无【答案】标记） | `1. A`、`1.C` |
| 选择题解析 | 行内格式 `【解析】` 紧跟答案 | `2. A 【解析】声音的传播...` |
| 大题答案 | 题号后直接跟小问或答案 | `5.(1)水平 调节平衡...` |
| 大题小问 | `(1)``(2)` 格式，可能多小问同行 | `(1)振动 空气 音色 (2)510 340` |
| 独立解析 | 部分大题有独立 `【解析】` 行 | `【解析】(1) 冰块的体积为...` |
| 章节标题 | `第X讲 标题` 格式 | `第一讲 声现象` |
| 小节标题 | `基础过关`/`能力提升`/`素养拔尖` | 布心中学特有 |
| 子课时 | `第 X 课时 标题` | `第 1 课时 质量 密度` |

## 开发过程

### Step 1: 文档分析

通过分析样本文档，识别出以下关键模式：

1. **无【答案】标记**: 与模板B/C不同，答案直接跟在题号后
2. **行内解析**: 选择题解析紧跟答案，格式为 `【解析】内容`
3. **多小问同行**: 大题小问可能连续出现在同一行，如 `(1)xxx (2)yyy`
4. **半角点号**: 使用 `1.` 而非 `1．`（全角点号）

### Step 2: 模板设计

基于模板C的结构，进行以下关键修改：

#### 2.1 正则模式调整

```python
# 选择题答案模式（支持行内解析）
CHOICE_ANSWER_PATTERN = re.compile(r"^(\d+)\.\s*([A-D])\b\s*(.*)")

# 行内解析提取
INLINE_ANALYSIS_PATTERN = re.compile(r"【解析】\s*(.*)$")
```

#### 2.2 多小问拆分逻辑

```python
# 处理可能包含多个小问的内容
sub_questions = re.findall(r'[(（](\d+)[)）]([^（(]*?)(?=[(（]\d+[)）]|$)', content)
if sub_questions:
    for sub_num, sub_content in sub_questions:
        # 分别处理每个小问
```

#### 2.3 垃圾行过滤

```python
GARBAGE_PATTERNS = [
    r"^第[一二三四五六七八九十\d]+讲",   # 第一讲、第二讲
    r"^第\s*\d+\s*课时",               # 第 1 课时

    r"^能力提升$",                      # 能力提升

    r"^答:略。?$",                      # 答:略。
    r"^答：略。?$",                      # 答：略。
]
```

## 关键问题与解决方案

### [问题1] 多小问同行识别

**现象**: 一行中包含多个小问，如 `6.(1)振动 空气 音色 (2)510 340`

**解决方案**: 使用正则拆分多个小问
```python
sub_questions = re.findall(r'[(（](\d+)[)）]([^（(]*?)(?=[(（]\d+[)）]|$)', content)
```

### [问题2] 半角点号兼容

**现象**: 文档使用 `1.` 而非 `1．`

**解决方案**: 正则同时支持半角和全角点号
```python
QUESTION_PATTERN = re.compile(r"^(\d+)[．.](.+)")
```

### [问题3] 行内解析提取

**现象**: 选择题解析紧跟答案，如 `2. A 【解析】声音的传播...`

**解决方案**: 在匹配选择题时同时提取行内解析
```python
choice_match = CHOICE_ANSWER_PATTERN.match(text)
if choice_match:
    remaining = choice_match.group(3).strip()
    inline = INLINE_ANALYSIS_PATTERN.search(remaining)
    if inline:
        analysis_text = inline.group(1).strip()
```

### [问题4] 输出格式标准化

**现象**: 输出格式需要符合 `Standard output format.md` 标准

**解决方案**: 
- 选择题有解析时：`f"{output_idx}．{letter}　解析：{analysis_lines[0]}\r"`
- 选择题无解析时：`f"{output_idx}．{letter}　\r"`
- 大题：`f"{output_idx}．\r"` + `f"答案：{answer_lines[0]}\r"`

### [问题5] 章节标题保留

**现象**: 用户需要保留章节标题（如"第五讲"、"第 1 课时"）以便检查多章节文档

**解决方案**: 
- 将章节标题从 `GARBAGE_PATTERNS` 移除
- 新增 `SECTION_PATTERNS` 识别章节标题
- 新增 `is_section_title()` 函数判断章节标题
- 章节标题作为类型 `section` 保存，不占用题号，原样输出
