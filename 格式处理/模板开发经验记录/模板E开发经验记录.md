## 开发过程

### Step 1: 文档分析

通过分析三个待清洗文件，发现它们属于**两种不同的格式**：

1. **限时训练 (二) 答案.docx**: 有 `【答案】` + `【详解】` → **可用模板B**
2. **高二限时训练1答案.docx**: 只有 `【答案】`，无详解 → **需要模板E**
3. **高二限时训练答案3.docx**: 只有 `【答案】`，无详解 → **需要模板E**

### Step 2: 关键设计决策

#### 决策1: 题号+答案组合格式识别

**问题**: 文档使用 `1．【答案】A` 或 `3【答案】B` 格式，题号和答案在同一行

**解决方案**: 使用组合正则匹配
```python
QUESTION_ANSWER_COMBO = re.compile(r"^(\d+)[．.]?\s*【答案】\s*([A-D])$")
```

#### 决策2: 大题答案续行收集

**问题**: 大题答案可能跨多行，如：
```
14．【答案】(1)2NA或1.204×1024     (2)1∶2      1∶1
(3)3        (4)5∶1
```

**解决方案**: 
1. 识别 `(1)` 开头的小问行
2. 向后查找连续的小问行 `(2)`, `(3)`...
3. 非小问行、非新题号行视为续行，追加到上一行

#### 决策3: 选择题与大题区分

**问题**: 如何区分 `1．【答案】A`（选择题）和 `14．【答案】(1)xxx`（大题）

**解决方案**: 检查【答案】后的内容
```python
if re.match(r'^[A-D]$', answer_text):
    # 是选择题
else:
    # 是大题
```

### Step 3: 垃圾行过滤

针对化学限时训练文档，新增以下垃圾行模式：

```python
GARBAGE_PATTERNS = [
```

## 问题与解决方案

### [问题1] 题号格式不一致

| 项目 | 内容 |
|------|------|
| **现象** | 有的题号有全角点号 `1．`，有的没有 `3` |
| **根因** | 文档格式不统一 |
| **解决** | 正则使用 `[．.]?` 使点号可选：`r"^(\d+)[．.]?\s*【答案】"` |
| **标签** | `[识别]` |

### [问题2] 大题答案续行丢失

| 项目 | 内容 |
|------|------|
| **现象** | 大题答案只提取了第一行，后续 `(2)(3)(4)` 丢失 |
| **根因** | 未实现向后查找内循环 |
| **解决** | 使用 while 循环收集续行，直到遇到新题号或小问编号 |
| **标签** | `[丢失]` |

### [问题3] 答案内容跨行

| 项目 | 内容 |
|------|------|
| **现象** | 长答案内容跨多行，如化学方程式 |
| **根因** | 答案内容较长，自然换行 |
| **解决** | 非题号、非【答案】、非小问编号的行视为续行，追加到上一行答案 |
| **标签** | `[截断]` |

## 模板特点

### 优势
1. 支持题号+答案组合格式（一行内完成）
2. 自动收集大题多小问答案
3. 支持答案内容跨行续接
4. 针对化学文档优化垃圾行过滤

### 局限
1. 仅支持 `【答案】` 标记格式
2. 不处理任何解析/详解内容
3. 假设选择题答案只能是 A-D 单字母
4. 输出格式只有答案，无解析
