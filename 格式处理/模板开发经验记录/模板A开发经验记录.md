## 开发过程

### Step 1: 文档分析

通过分析样本文档，识别出以下关键模式：

1. **章节标题模式**: `^\d+\s+[^0-9]` - 用于识别课文单元标题
2. **选择题模式**: `^\d+\.\s*([A-D])\b` - 简单选择题
3. **带小问选择题**: `^\d+\.\s*[(（](\d+)[)）]\s*([A-D]?)` - 如 7. (1)B
4. **大题模式**: `^\d+\.\s*(.+)` - 匹配题号后的所有内容
5. **小问模式**: `^[(（](\d+)[)）]\s*(.*)` - 匹配独立小问行

### Step 2: 关键设计决策

#### 决策1: 区分选择题和大题的关键逻辑

**问题**: `6. (1)锄豆...` 和 `7. (1)B` 都匹配 `QUESTION_WITH_SUB_PATTERN`，如何区分？

**解决方案**: 检查答案部分是否为字母 A-D
```python
if answer and re.match(r'^[A-D]$', answer):
    # 是选择题
else:
    # 是大题
```

#### 决策2: 多小问同行处理

**问题**: 一行可能包含多个小问，如 `(1)B (2)D (3)C A B`

**解决方案**: 使用正则提取所有小问
```python
MULTI_SUB_PATTERN = re.compile(r"[(（](\d+)[)）]([^（(]*?)(?=[(（]\d+[)）]|$)")
subs = MULTI_SUB_PATTERN.findall(remaining)
```

#### 决策3: 续行收集

**问题**: 答案内容可能跨多行（如第6题的示例描述很长）

**解决方案**: 使用 while 循环向后查找，直到遇到新题或小问

### Step 3: 测试验证

#### 正则测试

## 问题与解决方案

### [问题1] 选择题与大题区分

| 项目 | 内容 |
|------|------|
| **现象** | `6. (1)锄豆...` 被误判为选择题 |
| **根因** | QUESTION_WITH_SUB_PATTERN 同时匹配两种格式 |
| **解决** | 增加答案字母检查：`re.match(r'^[A-D]$', answer)` |
| **标签** | `[识别]` |

### [问题2] 多小问同行提取

| 项目 | 内容 |
|------|------|
| **现象** | `3. (1)B (2)D (3)C A B` 只提取了第一个小问 |
| **根因** | 正则只匹配第一个小问模式 |
| **解决** | 使用 `findall` 提取所有小问：`MULTI_SUB_PATTERN.findall()` |
| **标签** | `[丢失]` |

### [问题3] 答案续行丢失

| 项目 | 内容 |
|------|------|
| **现象** | 长答案只提取了第一行 |
| **根因** | 未实现向后查找续行 |
| **解决** | 使用 while 循环收集续行，直到遇到新题标记 |
| **标签** | `[截断]` |

## 模板特点

### 优势
1. 自动区分选择题和大题
2. 支持多小问同行格式
3. 自动收集答案续行
4. 保留章节标题作为分隔

### 局限
1. 仅支持 `(1)(2)(3)` 格式小问，不支持 `①②③`
2. 假设选择题答案只能是 A-D 单字母
3. 不处理【解析】标记（本文档无此格式）

## 章节选择功能

### 功能说明

模板A支持交互式章节选择，在清洗多章节文档时，用户可以选择只清洗特定章节。

### 使用方式

1. 运行 `main.py` 选择清洗文档
2. 模板会自动扫描并列出所有章节：
   ```
   📑 检测到 11 个章节：

   [1] 1 古诗词三首
   [2] 2 乡下人家
   [3] 阅读素养练习(一)
   ...

   [0] 清洗全部章节
   [a] 全选所有章节
   ```
3. 输入选择：
   - `0` - 清洗全部章节
   - `a` - 全选所有章节
   - `1,3,5` - 选择第1、3、5个章节
   - `1-3` - 选择第1到3个章节
   - `1,3-5,7` - 混合选择

### 多章节分别保存

当选择多个章节时，系统会询问是否分别保存：

```
💡 检测到选择了 3 个章节
是否将每个章节保存为单独文件？(y/n，默认n): y
```

选择 `y` 后，每个章节会保存为独立文件：
- `四下语文1-2答案2_已清洗1.docx` - 第1个章节
- `四下语文1-2答案2_已清洗2.docx` - 第2个章节
- `四下语文1-2答案2_已清洗3.docx` - 第3个章节

### 实现要点

- `scan_sections()` - 扫描文档中的所有章节标题
- `select_sections()` - 交互式选择章节
- `clean_document(return_sections_info=True)` - 返回章节信息用于分别保存
- 清洗范围根据选择的章节自动计算（从章节开始到下一个章节前）
