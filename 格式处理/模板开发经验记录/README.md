## 使用说明

### 开发新模板前

1. 阅读 [TEMPLATE_DEV_GUIDE.md](../格式模板库/TEMPLATE_DEV_GUIDE.md) 了解规范
2. 查看本文件夹中相似模板的经验记录
3. 预估可能遇到的问题（参考"按问题类型"索引）

### 遇到问题后

1. 先在本文件夹查找相似问题
2. 记录新问题到对应模板的经验记录文件
3. 更新本 README 的索引

### 模板开发完成

1. 创建 `{模板名}开发经验记录.md`
2. 更新本 README 的"记录列表"
3. 提交 git commit

## 模板开发最佳实践（基于经验总结）

### DO（推荐）

- ✅ 先分析样本文档结构，再写代码
- ✅ 从最接近的参考模板复制修改
- ✅ 每次修改后单文档测试
- ✅ 使用内循环收集多行内容
- ✅ 保留原始标记（【分析】【详解】等）
- ✅ 实现 `set_font_format` 统一字体
- ✅ 记录问题和解决方案

### DON'T（避免）

- ❌ 一次性写完整模板再测试
- ❌ 内循环停止条件过于宽泛
- ❌ 忽略章节编号等垃圾行
- ❌ 忘记更新 `__init__.py`
- ❌ **忘记在 `main.py` 的 `TEMPLATES` 列表中注册新模板**（仅更新 `__init__.py` 不够）
- ❌ 删除原始格式标记
- ❌ 跳过质量校验
- ❌ **收到待清洗样本后不检查解析续行（`①②③④`、`故本题选X。` 等）的收集逻辑**
- ❌ **修改代码后不亲自测试就报告用户"修改完成"**（必须先跑完测试闭环验证）
