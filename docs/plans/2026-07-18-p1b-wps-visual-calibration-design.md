## 1. 背景与问题

P1a 已能根据 `DocumentProfile.json` 1.1 的结构指纹生成候选文档族，但其输入主要来自 OOXML 和 `python-docx`。这类结构扫描可以发现图片节点，却不能证明 WPS 最终把图片显示成什么，也不能可靠区分以下两类媒体：

- 需要随题目进入 F1 的实验图、示意图和数据图；
- 必须排除的章节横幅、`对点训练`、`综合强化` 等装饰图片。

未来高二生物真实样本已经证明该缺口会影响动作计划：`作业1　细胞生活的环境.docx` 中，`综合强化` 是第 8 题和第 9 题之间的独立图片段，当前预演计划却把它算进第 8 题的媒体范围。按该图片的已确认精确内容统计，真实活页批次中出现 42 次，当前 42 次全部落入前一道 F1 动作。

因此，P1b 不能只做相似度阈值扫描。它必须增加 WPS 页面视觉真值，并以“是否可以安全使用同一套录入规则”作为人工分族口径。

## 2. 目标

P1b 完成以下闭环：

1. 使用 WPS 只读打开原题并导出 PDF，逐页转成固定尺寸 PNG；
2. 生成页面级机器事实，记录页面图像、文本块、图片块和版面坐标；
3. 将页面区域标记为题目正文、题目图片或应忽略内容，并绑定题号；
4. 把视觉区域与 OOXML 媒体、段落坐标及现有 F1 预演动作对齐；
5. 用人工确认的“录入规则等价”标签校准当前真实批次；
6. 生成整批审核门禁报告，明确哪些文档已确认、哪些媒体未知、哪些动作存在污染；
7. 所有结论继续保持旁路性质，不触发 WPS 录入。

## 3. 非目标与安全边界

- 不修改、保存、覆盖或写回任何原始 DOCX；渲染前后必须核对 SHA256。
- 不把 PNG、PDF、JSON 或 Markdown 写入原题目录。
- 不连接墨痕插件，不触发 F1/F2/F3/F4。
- 不修改 `wps_helper.py`。
- 不在 P1b 自动生成生产规则快照；规则快照属于 P1c。
- 不让 OCR 或视觉模型的低置信度猜测自动删除内容。
- 不把图片尺寸、宽高比或“空白段图片”作为单一删除依据。
- 不因同一图片哈希重复出现就直接认定其为装饰图；重复哈希只作为证据之一。

所有 P1b 输出固定包含：

```text
classification_mode = human_calibrated_advisory
automatic_rule_binding_enabled = false
production_execution_enabled = false
```

## 4. 方案选择

### 4.1 只做 WPS 窗口滚动截图

优点是直接反映当前屏幕；缺点是受窗口大小、缩放、滚动位置、菜单遮挡和分页停靠影响，难以稳定复现。只保留为人工临时取证方式，不作为批处理主链。

### 4.2 WPS 只读导出 PDF，再逐页转 PNG（采用）

WPS 负责排版渲染，PDF 固化分页和可见内容，PNG 供视觉审核。该方式不依赖窗口滚动，页面尺寸和坐标稳定，也便于重新生成和比较。

### 4.3 只分析截图，不与文档结构对齐

能够看懂页面，但不能把识别结果稳定映射回 WPS Range 和原生媒体关系，无法形成安全执行契约。不采用。

最终采用“双视图对齐”：WPS 页面渲染提供视觉事实，OOXML/WPS Range 提供结构和执行坐标。

## 5. 总体数据流

```text
原始 DOCX（只读）
  ├─ WPS 导出 PDF → 逐页 PNG → PageRenderManifest.json
  └─ OOXML / 现有预检 → DocumentProfile.json + ActionPlan.json
                         ↓
               页面区域与结构锚点对齐
                         ↓
                 VisualRoleReview.json
                         ↓
               P1bCalibrationReport.json
                         ↓
           Markdown 审核报告（仅由 JSON 派生）
```

原始 `DocumentProfile.json` 和 `ActionPlan.json` 不被覆盖。P1b 通过引用源 SHA256、页面哈希和动作序号建立旁路审核关系。

## 6. 输出契约

### 6.1 `PageRenderManifest.json`

只记录可重复验证的机器事实：

- schema 版本和生成器版本；
- 原始 DOCX 名称、绝对路径、SHA256；
- WPS 版本、PDF 页数和导出状态；
- 每页 PNG 路径、尺寸和 SHA256；
- 页面文本块、图片块及其边界框；
- PDF 页面与 WPS 页码的对应关系；
- 渲染前后原题哈希核对结果；
- 捕获失败、空白页、异常页面尺寸等问题。

图片路径使用相对于审核产物目录的路径，不在 JSON 中嵌入 base64。

### 6.2 `VisualRoleReview.json`

保存页面区域的角色判断和证据，不把模型推断伪装成机器事实。每个区域至少包含：

```json
{
  "region_id": "page-001-region-007",
  "page": 1,
  "bbox": [60, 185, 225, 225],
  "role": "exercise_label",
  "visible_text": "对点训练",
  "decision": "exclude",
  "belongs_to_question_ids": [],
  "confidence": 1.0,
  "evidence": ["wps_visible", "user_confirmed", "repeated_media"],
  "review_status": "confirmed"
}
```

角色枚举固定为：

| 角色 | 含义 | 默认处理 |
|------|------|----------|
| `document_banner` | 章节横幅、卷头装饰图 | 排除 |
| `document_title` | 作业标题、试卷标题 | 排除 |
| `score_instruction` | 分值和题号计分说明 | 排除 |
| `exercise_label` | 对点训练、综合强化等训练标签 | 排除 |
| `group_heading` | 题组、选择题、非选择题等结构标题 | 排除 |
| `question_text` | 题干、选项、小问 | 保留并绑定题号 |
| `question_media` | 题目对应图片、图表 | 保留并绑定题号 |
| `unknown` | 证据不足 | 阻断人工复核 |

`review_status` 只有 `draft`、`confirmed`、`rejected`。只有 `confirmed` 决策可以被整批门禁消费。

### 6.3 `P1bCalibrationReport.json`

记录：

- 人工真值定义：能否使用同一套 F1 录入规则；
- 每份文档的执行规则族标签和文档外形标签；
- P1a 阈值扫描结果及误拆、误并情况；
- 当前批次建议阈值及其适用范围；
- 代表样本、边界样本和异常样本；
- 每份文档的视觉审核覆盖率；
- 装饰内容进入动作的数量；
- 题目媒体未绑定或被误排除的数量；
- 未确认区域和阻断原因；
- 三个固定安全开关。

## 7. 规则族与文档外形

P1b 不按文件名、章节名或页面外观直接分生产规则族。当前试验假设为：

```text
执行规则族：science_core + biology_overlay（候选）
文档外形：worksheet / chapter_test
```

普通作业和章末检测可以有不同标题和分页，但只要顶层题号边界、题图绑定和排除规则相同，就应属于同一执行规则族。文档外形只作为解释性标签，不自动产生两套规则。

如果视觉审核发现某一类文档确实需要不同的 F1 边界、媒体绑定或小问处理，才将其拆为另一个人工真值族。

当前 52 份文档若最终都标为同一执行规则族，只能校准“同族文档不得被误拆”的批次阈值下界；由于缺少批次内反例，不能由此证明该阈值适合跨项目生产分族。报告必须把阈值标为 `batch_scoped`，不得升级成全局生产阈值。

## 8. 页面区域与结构锚点对齐

对齐优先使用可验证证据，不使用单一启发式：

1. 页面文字与 OOXML 段落的规范化文本匹配；
2. 图片媒体 SHA256 与 OOXML relationship ID；
3. 图片锚定段及其前后题号；
4. 现有动作的 `source_ref` 段落和媒体坐标；
5. 题干中的“如图、据图、下图”等引用；
6. WPS 页面上的空间邻接关系；
7. 用户确认的视觉角色。

同一图片可能位于空白段：

- 若图片位于题干附近，题干明确引用图片，并能绑定到唯一题号，可判为 `question_media`；
- 若图片承担章节横幅或训练标签角色，可判为排除角色；
- 若两种解释都可能成立，必须标为 `unknown`。

系统不得采用“所有空段图片都排除”或“所有题号之间图片都属于前题”的规则。

## 10. 整批审核门禁

P1b 批次可以标为“人工校准完成”必须同时满足：

1. 52 份真实活页全部渲染成功，页数和页面文件完整；
2. 52 份源 DOCX 渲染前后 SHA256 完全不变；
3. 所有页面区域均有角色，或明确列入人工复核；
4. 所有纳入结论的角色决策均为 `confirmed`；
5. 已确认装饰区域进入 F1 的数量为 0；
6. 已确认题目媒体被排除或未绑定题号的数量为 0；
7. 每个 F1 动作只包含对应题干、选项、小问、表格和题目媒体；
8. 顶层题号序列与人工抽查一致；
9. P1a 候选族与人工真值的误拆、误并均被报告；
10. 所有输出 JSON 可重新读取，Markdown 与 JSON 数量一致；
11. `automatic_rule_binding_enabled=false`；
12. `production_execution_enabled=false`。

这里的“装饰区域进入 F1 为 0”指审核后的 P1b 规则建议和动作覆盖层，不代表生产解析器已经自动消费这些规则。生产绑定仍需 P1c。

## 11. 错误与阻断处理

以下情况整份文档阻断，不生成可放行的审核结论：

- WPS 打开或导出失败；
- 导出页数为 0、出现空白页或页面尺寸异常；
- 原题 SHA256 发生变化；
- PDF、PNG 或 JSON 缺失、损坏或哈希不一致；
- 页面区域无法映射到唯一结构锚点；
- 图片可能是题图也可能是装饰图；
- OCR 文本和 WPS 可见内容存在关键冲突；
- 视觉结果与现有题号边界冲突；
- 发现 WPS `~$`、`.~` 等临时锁文件时误把它当作源文档。

失败文档进入中文人工复核队列；不得跳过坏文件后把剩余批次声明为完整通过。
