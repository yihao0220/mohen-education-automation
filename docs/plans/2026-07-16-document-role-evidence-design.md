# DOCX 题内角色证据层设计

> 状态：已实现并复核（2026-07-16）。角色分布已进入 P1a 文档族候选聚类与异常样本发现，但仍不直接控制生产 F1 选区。

## 目标

在不改变现有题块解析和 WPS 行为的前提下，为只读预检增加可解释的段落角色证据，解决“小标题只能靠精确文字匹配”的架构缺口。

```text
原始 DOCX（只读）
→ 原生 OOXML 段落特征（坐标真值）
→ Docling 语义标签（可选辅助证据）
→ 保守角色评分
→ DocumentProfile.roles（机器事实）
→ Markdown 角色候选审核视图
```

## 方案比较

### 方案 A：直接用 Docling 标签决定角色

实现快，但 Docling 元素不能稳定替代原始 Word 段落坐标；重复文本和拆分文本还可能造成错误映射。不采用。

### 方案 B：原生特征为主、Docling 为辅（采用）

原生扫描提取样式、粗体、字号、段落长度、编号形式、上下文和位置；Docling 标签只通过单调文本对齐加入辅助证据。所有判断输出分数、置信度和证据，不直接修改题块或排除段落。

### 方案 C：只扩充标题关键词

风险最小但继续过拟合具体文字，无法解决根问题。不采用。

## 第一版角色集合

- `document_title`：文档级标题；
- `section_heading`：顶层板块标题；
- `internal_heading`：题内短标题候选；
- `question_start`：明确题号起点；
- `option`：选择题选项；
- `body`：普通正文、材料或题干续行；
- `unknown`：证据不足，禁止强猜。

角色分类是预检信息，不是执行指令。尤其 `internal_heading` 第一版只进入人工审核候选，不自动从 F1 选区删除。

## 证据与评分边界

原生正向证据包括：Heading/标题样式、粗体占比、相对字号、短段落、无句末标点、中文章节编号、下一段为题号或正文。反向证据包括：明确题号、选项前缀、长文本、表格内容、公式或媒体承载。

Docling 的 `title`、`section_header`、`list_item`、`paragraph` 标签只增加辅助分，不覆盖原生反向证据。重复文本采用从前向后的单调对齐；无法唯一或顺序一致地对齐时不绑定。

每个结果至少包含：原始段落索引、角色、置信度、总分、证据列表、文本预览、Docling 标签（如有）。

## 错误处理与安全边界

- Docling 未安装、失败或无法对齐：原生分类继续，记录辅助证据缺失；
- 证据冲突或分数不足：输出 `unknown`；
- 原文件哈希变化：沿用现有硬阻断；
- `ActionPlan.execution_enabled` 继续固定为 `false`；
- 不修改 `question_core.py`、`subject_overlay.py`、`core_parser.py` 或 `wps_helper.py` 的生产行为。
