# 未来高二生物答案标准化清洗

## 现象

`tools/trim_future_biology_answers.py` 只完成了“课时对点练”前后的边界截取。派生文档仍保留题干、选项、题组标题、题干表格和“综合强化”装饰图，不是可直接录入 F2/F3/F4 的标准答案。

## 根因

把“删掉前缀废话”误当成了“答案清洗完成”。两个阶段的输入输出契约不同：截取阶段保留边界后全部原始结构；清洗阶段只能保留题号、答案、解析及其富文本对象。

## 改动点

- 新增 `tools/clean_future_biology_answers.py`，只读取 `答案/按课时截取/选必一答案|选必二答案`。
- 每道题标准化为 `N．` → `答案：` → `解析：`；无解析时显式补空解析。
- 删除题干、选项、题干表格、题组/题型标题和已确认的装饰媒体。
- 保留答案或解析区的图片、公式和富文本运行；连续的纯文本答案段合并到同一 `答案：` 段，避免 `(1)(2)(3)` 跨段时少识别小问。
- 审核时只把“题号到答案标记之前”当作题目区，避免解析里重复的小问号污染题目计数。
- 允许答案按实际有答案的小问定义 F4 项；例如题目包含 `(1)` 至 `(5)`，但 `(2)(3)` 是已给出的“实验目的/原理”，答案仅保留 `(1)(4)(5)`。
- 生成审核清单、审核状态和 `FutureBiologyAnswerCleanManifest.json`。

## 真实批次结果

- 输入 42 份，共 580 道题。
- 输出题号 580、答案 580、解析 580。
- 补空解析 46 道。
- 排除结构标题 50 处、装饰媒体 41 处。
- 保留答案/解析媒体 5 处。
- 输出中选项、题干表格、结构标题均为 0；42/42 份通过自动审核门禁。
- 清洗前后全部输入 SHA256 不变。

## 测试样例

- 合成样本覆盖题干、选项、表格、结构标题、跨段小问答案、答案图和解析图。
- 整批门禁要求 42/580/46/50/41/5 精确匹配，并验证 42 份审核状态可录入。
- 两个真实反例锁定“跨段 `(1)(2)(3)`”和“只有 `(1)(4)(5)` 需要答案”。

## 测试命令

```bash
.venv/bin/python -m pytest -q -p no:cacheprovider test_future_biology_answer_clean.py
.venv/bin/python tools/clean_future_biology_answers.py "/绝对路径/未来-高二-生物" --preflight-only --expected-source-count 42 --expected-question-count 580
```

## 结论

“按课时截取”只是中间输入，不再宣称为已清洗答案。专项清洗器已完成 42 份、580 题的离线开发和审核门禁；Mac 结论不替代 Windows WPS 的 F2/F3/F4 实际落位验证。

## 下次排查入口

1. 题数或缺解析数变化：先运行 `--preflight-only`，不要直接覆盖旧输出。
2. 仍有题干/选项：查 `QUESTION_PATTERN` 与答案标记数，不要在通用答案解析器里加批次特判。
3. 装饰图或答案图角色异常：查 `future_biology` 媒体哈希及其在题块中的位置，未知媒体不能猜测删除。
4. F4 数量异常：先对照清洗答案中实际的 `(n)` 项和 WPS 插件答案框，不按题干中所有小问号盲目补 F4。
