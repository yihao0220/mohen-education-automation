# 众美项目目录改名后答案上下文与审核状态失效

## 现象

项目总目录从 `众美-高三-语文-王逸豪` 改为 `众美-高三-语文` 后，28 份文言文清洗结果仍在，但审核状态 JSON 记录的答案绝对路径已失效。重新审核时，双篇文档 `07《庖丁解牛》《烛之武退秦师》` 只识别出第一篇的 5 个答案块，并误报第二篇翻译题小问数量不一致。

## 根因

- `shared_core/answer_core.py` 用完整项目目录名识别众美文言文上下文；去掉人员后缀后，双篇标题边界、重复题号按出现顺序映射等专项规则没有启用。
- 众美专项工具和真实样本测试把旧项目绝对路径写成默认值，目录改名后会找不到样本或跳过测试。
- 审核状态与答案文件的绝对路径、大小和修改时间绑定，父目录改名后旧状态必须刷新；不能只移动 JSON 后继续放行。

## 改动点

- 将众美项目上下文标记收敛为稳定前缀 `众美-高三-语文`，同时兼容新旧目录名。
- 同步众美文言文、对点练清洗、阅读组合并、审核状态工具及真实样本测试的默认路径。
- 对当前目录下 28 份文言文清洗答案重新执行题答检查并刷新审核状态，不改用户现有文件名。

## 测试样例

- 双篇重号：`07《庖丁解牛》《烛之武退秦师》` 应解析为 9 个题目块和 9 个答案块，不得只保留第一篇。
- 单小题翻译：`10《与妻书》《促织》` 的翻译题仍保留 `(1)` 并使用 F4。
- 正式整批：28 份文言文答案应映射为 154 个题目块和 154 个答案块，审核问题为 0。

## 测试命令

```powershell
python -m pytest -q test_zhongmei_classical_chinese_answer_cleaner.py test_duplicate_answer_question_ids.py test_review_duplicate_answer_question_ids.py test_zero_error_pipeline.py test_zhongmei_chinese_answer_cleaner.py test_zhongmei_reading_answer_merge.py test_zhongmei_chinese_answer_input.py test_zhongmei_chinese_workflow.py test_zhongmei_review_status.py test_shared_core_flow.py
```

## 结论

- 完整回归 `91 passed`，另有 `591 subtests passed`；仅有 15 条既有 UTC 弃用警告。
- 正式文言文答案重新审核结果：28/28 approved，154 个题目块与 154 个答案块对应，审核问题 0。
- 当前项目总目录和清洗结果文件名均保持用户改名后的状态，没有恢复旧名称。

## 下次排查入口

如果项目目录再次改名后出现双篇答案缺块或真实样本测试被跳过，先检查 `shared_core/answer_core.py` 的项目标记是否仍使用稳定前缀，再搜索 Python 文件中的旧绝对路径，最后重新刷新审核状态。

## 2026-07-18 后续修订

本记录反映的“绝对路径/mtime 变化导致状态失效”已由 [审核状态跨机携带与 Windows 打包](../答案录入/2026-07-18-审核状态跨机携带与Windows打包.md) 解决。现在状态只按 `size + SHA256` 校验内容；但旧版状态没有 SHA256，仍必须重新生成，不能静默放行。
