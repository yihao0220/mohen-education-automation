# 材料题答案按原题范围重写为小问结构

## L1 摘要
- 模块：格式转换
- 日期：2026-04-21
- 状态：已完成
- 结论：答案清洗阶段现在会参考原题 docx 的材料题范围；如果题目是“回答3-5题 / 完成6-8小题”这类结构，清洗后的标准答案会重写成以首题号为主问题、后续题号折叠为小问的格式，例如 `3．(1)D / (2)C / (3)C / 解析：...`。

## L2 现象与根因
- 现象：原始答案文档里材料题常写成 `3．D / 4．C / 5．C` 三道独立答案，但题目录入侧已经把“材料+3-5题”视为一个题块；结果清洗后的答案结构与原题结构脱节，后续录入无法按大题小问方式稳定处理。
- 根因：原有清洗流程只解析答案文档自身，不看对应题目文档；`shared_core.map_answers()` 也只做题号对齐和审核，不会把材料题范围真正回写到清洗产物中。

## L4 深层备注
- 风险：
  - 当前“范围优先”依赖材料行里出现明确的 `3-5题 / 3～5小题 / 3至5题` 这类表达；如果题干只写“回答下面问题”而后面又是多个独立题号，仍然依赖现有题块边界判断。
