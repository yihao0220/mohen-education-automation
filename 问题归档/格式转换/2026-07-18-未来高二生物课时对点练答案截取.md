# 未来高二生物按“课时对点练”截取答案

## 现象

`未来-高二-生物` 的部分教师版答案在真正答案前保留了较长的教材或题目内容。业务边界已经明确：扫描到“课时对点练”后，其后内容才属于要录入的答案；此前内容全部丢弃。

同一批文档存在两种边界表示：

- `选必一答案` 使用 Word `INCLUDEPICTURE` 字段引用 `课时对点练新.TIF`，正文文本层没有标题。
- `选必二答案` 使用可见文字 `课时对点练　[分值：100分]`。

## 根因

普通段落文本扫描只能命中第二种表示，无法识别图片字段；若直接按可见文字批量删除，会漏掉 `选必一答案`。同时答案目录里还有章末、模块、主干、微专题等其他文档族，不能因为没有标记就猜测截取位置。

## 改动点

- 新增 `tools/trim_future_biology_answers.py`，只扫描 `选必一答案`、`选必二答案` 的有效 DOCX，并排除 WPS 锁文件。
- 整批写入前先预检所有文档；同时识别图片字段和可见文字两种边界。
- 图片字段边界删除标记段及之前内容；可见文字边界只删除“课时对点练”前缀，保留同行 `[分值：100分]`。
- 无标记的其他文档族明确跳过；重复标记、表格内标记、同行未知内容、标记后缺少答案或解析时整批停止。
- 输出固定为原答案之外的 `答案/按课时截取/`，保留源相对目录，不覆盖原答案；清单记录源/输出 SHA256。
- 新增 `test_future_biology_answer_trim.py`，并接入 `.github/workflows/p1b-windows.yml` 的 Windows 离线回归。

## 真实批次结果

- 有效答案 DOCX：73 份；另有 4 个 WPS 锁文件被排除。
- 命中并生成派生答案：42 份，其中图片字段 24 份、可见文字 18 份。
- 无标记并跳过其他文档族：31 份。
- 保留答案标记 580 个、解析标记 534 个。
- 原答案哈希全部不变；42 份输出均能被 `python-docx` 打开。
- 除 `word/document.xml` 外，DOCX 包内媒体、关系、样式等成员与源文件逐字节一致。

## 测试样例

- 图片字段标记：删除标记图片段和前部废话，保留后续分值、答案与解析。
- 可见文字标记：删除标题文字，保留同行分值和后续内容。
- 混合批次：仅处理有标记的文档，跳过无标记文档，忽略锁文件。
- 歧义门禁：重复标记和表格内标记均在创建输出目录前阻断。
- 包完整性：除正文 XML 外，其余 DOCX 成员保持不变。
- 真实批次锁定：73 / 42 / 24 / 18 / 31 的批次数量作为 Mac 定向回归；Windows runner 没有真实业务目录时只跳过这一条，仍运行全部合成测试。

## 视觉复核

42 份派生答案均使用 macOS Quick Look 生成连续预览，并以 21 张双文档联系表逐一目检；中文、图片、表格、公式均可见，未发现标题残留、截断、重叠或缺图。LibreOffice 对旧式中文字体/图片字段的渲染不可靠，不作为本批视觉结论。

Quick Look 仅是开发预览，固定不是 WPS 分页真值。GitHub Windows runner 也不安装 WPS，因此只能证明代码可在干净 Windows Python 环境安装和运行，不能替代生产机的 WPS 插件录入验证。

## 结论

“课时对点练”已成为本批唯一自动截取边界。脚本完成了不覆盖原答案的 42 份派生输出，并通过结构、哈希、包完整性、离线单测和 Mac 视觉复核。实际 F2/F3/F4 落位仍需在 Windows WPS 生产机上抽样后再整批执行。

## 下次排查入口

1. 批次数量变化：先运行 `--dry-run --expected-source-count 73 --expected-matched-count 42`。
2. 图片标题漏识别：检查 `word/document.xml` 的 `w:instrText` 是否仍包含 `课时对点练`。
3. 可见文字同行结构变化：检查标记后是否仍只有分值；未知内容应继续阻断，不要放宽为任意删除。
4. 输出缺图或格式变化：比较源/输出 DOCX 的 ZIP 成员及非 `word/document.xml` 字节。
5. WPS 录入乱位：转到 `答案录入/answer_input.py` 与 `shared_core/answer_core.py`，不要在截取脚本里补按键逻辑。
