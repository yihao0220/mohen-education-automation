# 清艳五年级数学答案按题型格式化与复合小题号

## L1 摘要
- 模块：格式转换 / 答案录入
- 日期：2026-06-28
- 状态：已完成
- 结论：清艳五年级数学答案整理不能机械输出 `【答案】+【解析】`。应根据题目文档决定答案区内容：填空题只放空内答案，计算题把完整列式过程放入答案区；多小问显示为 `3．（1）`、`3．（2）` 这类复合题号。

## L2 现象与根因
- 现象 1：整理版把第 3 题两个小问展开为第 3、4 题，导致题号与题目文档不一致。
- 现象 2：计算题答案区只有最终结果，完整列式过程放在解析区，不符合用户“直接给过程”的录入需求。
- 现象 3：答案解析器遇到 `3．（1）` 这类复合题号时，旧逻辑只能识别主编号 `3`，实际录入会把 `（1）` 一起选入答案。
- 现象 4：解析中的编号列表，如 `1.第一名`、`2.第二名`，会被误判为新答案块。

## L5 追加反馈：空解析与小问题号
- 现象：用户截图中第 3 章第 1 题有 `(1)(2)` 两个小问，但整理版显示成 `1．` 与 `2．`；第 1 小问后还出现了无内容的 `解析：`。
- 根因：答案文档按答案顺序重排时没有对齐题目文档的小问结构；复合小题号补写后又额外追加了一个 `．`；空解析占位未按“无解析不输出解析二字”的规则清除。
- 修复：
  - 按题目文档小问槽位对第 3 章、第 8 章安全重标号，输出 `1．（1）`、`1．（2）` 这类题号。
  - 删除无内容的 `解析：` 段；如果 `解析：` 后紧跟表格，则保留为表格解析引导。
  - 用 DOCX XML 层修复 `1．（1）．` 这类多余点号，避免重写段落样式。
- 验证：
  - 第 3 章开头已为 `1．（1）=15×...`、`1．（2）=40×...`。
  - `compound_extra_dot=0`，`empty_analysis_without_following_table=0`。
  - `python -m unittest test_compound_answer_ids.py test_qingyan_math_workflow.py` 通过。
