# VSCode右上角运行入口配置

## L1 摘要
- 模块：题目录入
- 日期：2026-04-20
- 状态：已完成
- 结论：已为 VSCode 新增 `launch.json`，可直接通过右上角运行按钮启动根目录总控入口或三个旧入口；同时修正了过时的 Python 分析路径。

## L2 现象与根因
- 现象：用户不想再手动切到 PowerShell 执行脚本，希望在 VSCode 里点右上角直接运行。
- 根因：工作区没有 `.vscode/launch.json`，且 `settings.json` 里的 `extraPaths` 仍指向历史目录 `core/`、`modules/`。

## L3 改动与验证
- 改动文件：
  - `.vscode/launch.json`
  - `.vscode/settings.json`
- 核心改动：
  - 新增运行配置：
    - `运行当前 Python 文件`
    - `墨痕教育：总控入口`
    - `墨痕教育：题目录入（旧入口）`
    - `墨痕教育：答案格式清洗（旧入口）`
    - `墨痕教育：答案录入（旧入口）`
  - 将 `python.analysis.extraPaths` 修正为当前真实目录：
    - `shared_core`
    - `墨痕快刀`
    - `格式处理`
    - `答案录入`
- 测试命令：
  - `Get-Content -Raw ".vscode/launch.json" | ConvertFrom-Json | Out-Null`
  - `Get-Content -Raw ".vscode/settings.json" | ConvertFrom-Json | Out-Null`
- 结果：
  - 两个 JSON 配置文件都通过了解析验证

## L4 深层备注
- 右上角默认运行哪个配置，取决于你在 VSCode 顶部选择的运行配置。
- 如果你只想平时点一次就走主流程，建议固定选择 `墨痕教育：总控入口`。
- 2026-04-20 补充：曾出现 VSCode 误用 `课表网站/backend/venv` 导致 `ModuleNotFoundError: No module named 'docx'`。
  - 已通过 `.vscode/settings.json` 的 `python.defaultInterpreterPath`
  - 以及 `.vscode/launch.json` 每个配置显式 `python`
  - 双重固定为 `${workspaceFolder}\\.venv\\Scripts\\python.exe`
  - 排查命令：`& ".\\.venv\\Scripts\\python.exe" -c "import docx; print('docx OK')"`
