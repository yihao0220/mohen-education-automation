# 全部表格题改为原生表格整块录入

## 现象

- 题目录入日志出现 `📋 [表格转文本] 行40-75 使用纯文本临时录入`。
- 用户确认表格无需序列化；应直接框选题干、原生表格和选项后按 F1，且规则适用于全部文档。

## 根因

- `墨痕快刀/core_parser.py` 仍保留“按 `.docx`/WPS 重建纯文本表格 → 创建临时文档 → F1”的历史兜底。
- 原生表格策略只对白名单覆盖层生效，普通学校、学科或未命中覆盖层的文档仍会走旧链路。

## 改动点

- 删除表格转纯文本、临时文档选区及相关运行链。
- 题块含表格时只记录 `[原生表格]`，最终选区始终来自原题文档的题干首端到题块末端。
- `should_use_native_table_input_for_context()` 改为全局策略，所有文档均返回原生表格录入。
- `AGENTS.md` 的跨项目规则同步改为“题干 + 原生表格 + 选项整块录入”。

## 测试样例与命令

- 普通语文、数学文件名且无覆盖层时，原生表格策略仍为 `True`。
- 运行链源码中不得再出现 `[表格转文本]` 或 `_select_text_via_temp_document`。

```powershell
python -m pytest -q test_table_input_format.py
```

## 结论

- 表格题的全局唯一录入方式现为：直接选中原题中的题干、表格和选项，再触发 F1。
- 2026-04-20、2026-04-21 的表格转文本方案均作为历史记录保留，但已不再生效。

## 下次排查入口

- 选区或日志异常：`墨痕快刀/core_parser.py` 的 `_range_contains_table()` 与 F1 前选区逻辑。
- 全局策略回归：`test_table_input_format.py`。
