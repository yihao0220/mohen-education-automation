# DOCX题内角色证据层

## L1 摘要

- 模块：题目录入 / 共享内核
- 日期：2026-07-16
- 状态：已完成并复核（只读预检）
- 结论：`DocumentProfile` 1.1 已能按结构证据区分文档标题、顶层板块标题、题内小标题、题号起点、选项和正文，并输出置信度与逐条证据。该结果目前只供画像、审计和后续分族使用，固定不自动删除段落，也不改变生产 F1 行为。

## L2 现象与根因

### 现象

- 题内小标题主要依赖“高考专练”“典题专练”等精确词表，文字稍变就可能失效。
- 只看短文本或 Docling 的标题标签又会把公式、解析短句、真正题号误判为标题。
- 当前需要先获得可解释的结构证据，不能直接让新分类结果控制 WPS 选区。

### 根因

- 既有预检画像保存了段落与结构摘要，但没有统一的段落业务角色模型。
- 原生 DOCX 特征和 Docling 标签之间没有稳定对齐及冲突优先级。
- “识别角色”和“自动排除标题”此前容易被当作同一步，缺少安全隔离。

## L3 改动点

### `shared_core/document_roles.py`

- 新增 1.0 角色模型：`document_title`、`section_heading`、`internal_heading`、`question_start`、`option`、`body`。
- 提取样式、标题样式、粗体比例、最大字号、段落长度、句末标点、中文序号前缀、题号、选项、媒体和公式等原生特征。
- 将 Docling 文本项与原生段落做单调精确对齐；无法对齐时不强行映射。
- 采用保守加权分类，输出 `role`、`confidence`、`evidence`；题号和选项强信号不能被 Docling 覆盖。
- 题内小标题必须至少具备一种真实标题信号，避免把短公式或短解析句当标题。

### `shared_core/document_preflight.py`

- `DocumentProfile` schema 从 1.0 升级为 1.1，并加入 `roles`。
- 结构指纹加入角色计数，已被 P1a 文档族候选聚类与异常样本发现使用。
- 人工 Markdown 增加角色统计、Docling 对齐情况、标题候选及命中证据。
- `ActionPlan` 仍为 1.0，固定 `mode=preview_only`、`execution_enabled=false`。

### 测试与黄金基线

- 新增题内标题与题号共存用例，验证标题能识别、题号不能被覆盖、动作计划不变。
- 新增 Docling 旁证冲突用例，验证第三方标签只能增强证据，不能覆盖强原生结构。
- 更新 pytest-regressions 黄金 JSON 和三份真实预检基线。

### 实测中修正的误判

- 初版仅凭“短文本 + 无句末标点”会把公式与解析短句判为题内标题。
- 修正后，题内标题必须同时具有标题样式、较高粗体比例、中文题内序号前缀或 Docling 标题标签之一。
- 顶层中文序号标题优先于普通标题判断；首个无顶层序号的 Heading 可判为文档标题。
