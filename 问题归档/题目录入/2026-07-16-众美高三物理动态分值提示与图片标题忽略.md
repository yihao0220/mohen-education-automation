# 众美高三物理动态分值提示与图片标题忽略

## 现象

- 各课时的分值范围不同，既有 `[1~8题，每题4分]`、`[9~11题，每题6分]`，也有 `[7题6分]`、`[6分]`、`[分值：54分]` 等紧凑写法。
- “基础落实练”“能力综合练”没有文字层，而是宽幅图片；图片与对应分值提示位于同一个 DOCX 段落。
- WPS COM 读取“能力综合练”图片所在段时，会在可见分值文字前增加图片文本标记 `/`，真实字符串为 `"/　[9~11题，每题6分]\r"`；只用纯 DOCX 文本测试无法发现这一差异。
- 首次修复后用户再次录入仍出现旧行为；运行中的两份 `墨痕快刀/main.py` 均启动于 22:17:44，而 `config.py` 修正规则写入时间为 22:29:00，旧进程不会自动重新导入 Python 模块。
- 如果只维护固定数字白名单，换课时后会把分值提示或装饰图片带入 F1 选区。

## 根因

理科障碍项原先偏向固定格式，没有把“题号范围、单题数量、每题分值、总分”统一建模。首次修正规则仍假设分值提示从左方括号开始，没有覆盖 WPS 给段首图片生成的 `/` 文本标记，因此最终 F1 Range 在第 8 题结束时会吞入标题图和分值行。规则修复后的再次失败并非新代码仍有同样缺陷，而是用户继续使用修复前已经启动的两个 Python 进程；Python 进程不会自动重新加载磁盘上的 `config.py`。图片标题本身无法用文字关键词匹配，但其锚定段落包含可识别的分值提示，因此应按整段排除，不能按图片内容做 OCR 或误删普通物理示意图。

## 改动点

- 在 `墨痕快刀/config.py` 增加通用 `SCIENCE_SCORE_CUE_PATTERN`，覆盖范围式、离散题号式、紧凑式和总分式提示，并允许方括号前出现一个可选的 WPS 图片标记 `/` 或 `／`。
- 将该规则加入理科 `obstacles`，F1 扫描和最终选区遇到对应段落时整段排除，段内的装饰图片随之排除。
- 工作台题答键归一化时去掉题目文件名前缀 `【空白试卷】`，保证众美题答自动配对。
- 回归测试不仅断言 `is_obstacle()` 命中，还完整检查 `should_ignore_inline_obstacle()` 不会因为第 8 题带图而再次放行这个分值障碍。
- 结束两份早于规则更新时间启动的 `墨痕快刀/main.py` 旧进程；保留答案录入进程不动。项目虚拟环境和系统 Python 的新进程都确认加载同一份 `墨痕快刀/config.py`。
- 未扩展到“尖子生选练”等用户未指定图片标题，避免误删未知题内媒体。

## 测试样例

- 规则样例：`[1~8题，每题4分]`、`[9~11题，每题6分]`、`[1、2题，每题4分]`、`[7题6分]`、`[6题6分]`、`[6分]`、`[分值：54分]`。
- WPS 真实文本样例：`/　[9~11题，每题6分]\r`，并带 1 个 `InlineShape`。
- 反例：`6.(10分)一道正常物理题` 仍是正常题号，不命中障碍项。
- 真实批次检查到“基础落实练”图片哈希重复 62 次、“能力综合练”图片哈希重复 27 次，均锚定在分值提示段落；题内物理示意图不按图片尺寸全局删除。
- 截图对应原题在 WPS 中第 8 题原始跨度为 36–41 段；第 41 段命中障碍后，最终跨度为 36–40 段，末段是 D 选项，跑道题图仍保留。
- 新进程按正式完整分支复验：第 41 段 `is_obstacle=True`、`should_ignore_inline_obstacle=False`，因此执行层确实提前截断，而不是只在扫描日志中识别。
- WPS 只读扫描全部 87 份原题：192 个分值段全部命中，漏判 0、打开失败 0、原题 SHA256 变化 0。

## 测试命令

```powershell
python -m unittest test_zhongmei_physics_workflow.py
python -m unittest test_zhongmei_physics_workflow.py test_future_physics_workflow.py test_shared_core_flow.py test_subject_detection.py test_zhongmei_review_status.py
```

结果：真实 WPS 文本失败用例修复前稳定失败、修复后通过；完整二次放行分支测试通过；专项测试 8 条通过，相关回归合计 25 条通过。两种 Python 解释器的新进程均从项目路径加载规则并命中真实字符串。

## 结论

分值数字变化不再需要逐份补白名单；两类图片标题及 WPS 产生的前导 `/` 通过“分值提示段落整段排除”稳定忽略。第 8 题 F1 Range 不再带入“能力综合练”和 `[9~11题，每题6分]`，原题保持只读不变。代码更新后必须退出旧的常驻录入进程再启动，否则仍会运行内存中的旧规则。

## 下次排查入口

1. 先查看 `墨痕快刀/config.py` 的 `SCIENCE_SCORE_CUE_PATTERN`。
2. 再查看 `墨痕快刀/core_parser.py` 的 `is_obstacle()` 和最终 F1 选区截断逻辑；排查时必须记录 WPS `Range.Text` 的 `repr()`，不要只看 `python-docx` 文本。
3. 如果磁盘代码已修复但控制台日志仍无对应命中，先比较 `main.py` 进程启动时间和规则文件修改时间，并确认是否同时运行了多个入口进程。
4. 若出现新图片标题，先确认其锚定段落是否有稳定文字线索，不要直接按图片尺寸删除。
