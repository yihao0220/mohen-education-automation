# DOCX文档族分组与异常样本发现P1a

## L1 结论

- 日期：2026-07-16
- 模块：题目录入 / DOCX 只读预检旁路
- 状态：已完成并复核（只读建议模式）
- 结论：系统已能从一批 `DocumentProfile.json` 1.1 生成可解释的候选文档族、代表样本、异常候选、未决单例和人工复核队列；结果不绑定生产规则，不触发 WPS。

## L2 现象与根因

### 现象

- 单文档画像和段落角色已经存在，但系统仍不能回答“一批文档实际包含多少种结构”。
- 人工需要逐份寻找代表样本和特殊文档，无法在真实录入前优先检查最有信息量的样本。
- 如果直接让 P0 角色结果排除标题，会绕过文档族阈值和审核门禁，误删风险过高。

### 根因

- 既有 `fingerprint` 只有单文档事实，没有批次相似度、分组和离群判断。
- 绝对段落数、题目数和媒体数受章节长度影响，不能直接作为文档族身份。
- 普通连通分组会发生 A 像 B、B 像 C、但 A 不像 C 的链式误并。

## L3 改动点

### `shared_core/document_families.py`

- 只读取 Profile 1.1，不重新解析或写回原始 DOCX。
- 将角色分布、动作/标题/媒体/公式密度、结构存在性和低权重文档规模归一化。
- 使用六项可解释加权相似度，每对文档保留分项分数、权重和贡献。
- 使用完全链接和 `0.78` 保守阈值形成候选文档族，防止链式误并。
- 多成员族使用 medoid 选择代表样本。
- 批次至少 3 份且单例最高近邻相似度 `< 0.60` 时，才标为异常候选；其余单例只标为待人工确认。
- 置信度 `< 0.75` 的标题候选、Profile warning/error 和未知角色进入人工复核队列。
- 报告固定 `classification_mode=advisory_only`、`automatic_rule_binding_enabled=false`、`production_execution_enabled=false`。

### `tools/analyze_document_families.py`

- 支持输入一个或多个 Profile JSON 文件或目录。
- 目录只递归发现 `*_DocumentProfile.json`，稳定排序并去重。
- 输出 `DocumentFamilyReport.json` 机器真值及其派生 Markdown。

### 回归资产

- 新增 `test_document_families.py`，覆盖完全链接、medoid、异常边界、确定性、坏输入、重复冲突和真实基线。
- 新增 `回归样本/文档族分析/DocumentFamilyReport.json/.md`。
