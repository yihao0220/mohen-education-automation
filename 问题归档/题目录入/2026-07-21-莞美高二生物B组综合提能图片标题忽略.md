# 莞美高二生物 B组综合提能图片标题忽略

## 状态

- 莞美专用覆盖层、专项测试和真实样本离线验证已完成。
- 原题全程只读，未覆盖、保存或生成题目副本。
- Windows WPS 真实 F1 选区和插件按键仍待生产机验证。

## 现象

`第1节　细胞的生活环境-排版终稿.docx` 的第9题与第10题之间有一张无文字层的“B组 综合提能”横幅图片。旧规则没有识别莞美排版终稿所属的生物覆盖层，离线题块把图片所在第41段并入第9题，导致F1选区可能带入该小标题。

## 根因

1. 图片标题没有可供正则匹配的DOCX文字，只能从媒体对象和上下文识别。
2. 现有 `future_biology` 覆盖层要求文件名包含“作业”或“检测试卷”，莞美文件名只含“排版终稿”，因此不会命中。
3. 目标图片哈希不在已确认的装饰媒体清单中，离线分块无法将其作为题间边界。

## 改动点

- 在 `shared_core/subject_overlay.py` 新增独立的 `guanmei_biology` 覆盖层：
  - 仅匹配文件名含“细胞的生活环境-排版终稿”且正文至少命中2个生物关键词的文档；
  - 将“B组 综合提能”图片 SHA256 `9cf693c46d970beea342457d6e13783cb9cc45c229f988a725f1f7e387f52463` 登记为 `exercise_label`；
  - WPS执行层只在“空白短横幅位于两道顶层题之间”时将其作为排除边界。
- 新增 `test_guanmei_biology_question_input.py`，覆盖项目识别、精确哈希边界、WPS上下文反例及真实DOCX验证。
- 未修改通用图片尺寸规则，未将莞美哈希加入未来生物覆盖层，避免跨学校误删普通题图。

## 测试样例

- 目标图片：`word/media/image6.png`，原始尺寸720×162，显示约170×42，内容为“B组 综合提能”。
- 原题SHA256：`8f520a0affabf80bcb28a944428aee381f2a0859a620541853011bed8e933ffb`。
- 修复前：第9题跨度36–41，媒体段为36、41；第41段是目标标题图。
- 修复后：第9题跨度36–40，仍保留第36段题图；第10题跨度42–45，仍保留第43段题图。
- 反例：图片段含“如图所示”文字，或下一段不是新题号时，不作为题间装饰标题排除。

## 测试命令

```powershell
python -m unittest -v test_guanmei_biology_question_input.py
python -m pytest -q test_guanmei_biology_question_input.py test_future_biology_question_input.py test_shared_core_flow.py
```

结果：专项4项通过；莞美、未来生物和共享内核相关回归共15项通过。原题测试前后SHA256一致。

## 结论

离线分块已不再把“B组 综合提能”标题图并入第9题，同时第9题和第10题的真实题图保持原样。该结论证明共享分块和WPS边界判定输入符合预期，但不等同于Windows WPS生产按键验证。
