# 众美答案小标题不再录入解析

## 现象

- F3 录入上一题解析时，会把下一板块的标题一并选中。
- 典型污染包括“一、专项训练”“(二)图文解读题”“二、综合训练”“阅读下面的文字，完成文后题目。”以及分析评价、逻辑推断、论证特点等小标题。
- 用户用 `【】` 标示需忽略的语句，方括号本身不是文档内容。

## 根因

- 众美专项清洗会保留标题供人工阅读。
- 共享答案解析器过去只在遇到下一题题号时结束上一题，夹在两题之间的标题因此被并入上一题解析。
- 旧版完整阅读组合并后，少数标题还会粘在上一段解析正文末尾，仅靠“标题独立成段”判断仍会漏掉。

## 改动点

- `shared_core/answer_core.py` 增加用户指定标题的精确边界规则，兼容半角/全角括号、空格和句号。
- 众美文档额外复用专项清洗器的标题形态：一级训练标题、二级小标题、阅读提示均作为答案块边界。
- 遇到标题时立即结束当前答案块；标题仍保留在 DOCX 中，不进入答案或解析内容。
- 对旧合并文档中粘在解析末尾的已知标题，按当前文档真实标题集合做尾部剥离，避免用宽泛正则误删普通解析列点。
- 不改变显式小问 F4、普通答案 F2、完整阅读组连续 F4 后一次 F3，以及未来高二物理整段 F2 例外。

## 测试样例与命令

```powershell
python -m pytest -q test_zhongmei_chinese_answer_input.py
python -m pytest -q test_zhongmei_chinese_answer_input.py test_zhongmei_chinese_workflow.py test_zhongmei_reading_answer_merge.py test_zhongmei_review_status.py test_zhongmei_chinese_answer_cleaner.py test_shared_core_flow.py test_zero_error_pipeline.py test_future_physics_workflow.py
python -m pytest -q test_future_physics_workflow.py -k "AnswerInput"
python tools/zhongmei_review_status.py
```

- 专项回归：13 passed，34 个 subtests 通过。
- 组合回归：72 passed，517 个 subtests 通过；未来物理完整文件仍有历史已知的2个共享映射失败，本次未新增。
- 未来物理 F2 录入例外：2 passed。
- 真实44份答案：修复前广义标题扫描发现53个污染答案块；修复后标题污染为0，44份审核状态全部 `approved`。

## 结论

- 众美答案中的训练标题、题型小标题和阅读提示不再进入 F2/F3/F4 选区。
- 标题保留在清洗文档中，不影响人工阅读；答案解析边界以标题前一段为止。
