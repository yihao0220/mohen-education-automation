# 众美答案自动审核状态补齐与门禁实时刷新

## 现象

- 打开众美 `对点练案N_已清洗.docx` 直接录入时提示：`❌ 已拦截：系统尚未放行当前答案文档，拒绝直接录入。`
- 44 份已清洗答案均缺少同名 `_审核状态.json`，因此安全门禁按设计拒绝录入。

## 根因

- 众美专项清洗和完整阅读组合并只改写 DOCX，没有运行题答自动检查，也没有生成与答案文件签名绑定的审核状态。
- `answer_input.py` 进入范围菜单前只读取一次门禁状态，外部补齐状态后仍可能继续使用旧结果。

## 改动点

- 新增 `tools/zhongmei_review_status.py`：先配对原题与已清洗答案，运行共享审核，再统一写入 `approved` 状态。
- 任何一份存在高风险错误时整批停止，避免部分文档被误放行。
- 众美批量清洗、完整阅读组永久合并完成后自动刷新审核状态。
- 答案录入在选择“全部录入”或“自定义范围”时重新读取当前状态，并显示门禁返回的具体原因与状态文件路径。
- 对现有 44 份已清洗答案执行真实复核，生成 44 份状态文件。

## 测试样例与命令

```powershell
python -m pytest -q test_zhongmei_review_status.py test_zhongmei_chinese_answer_input.py
python tools/zhongmei_review_status.py
```

- 聚焦测试：16 passed，另有 21 个 subtests 通过。
- 跨项目相关回归：72 passed，另有 623 个 subtests 通过。
- 真实文档：44 份已清洗答案、44 份状态文件、44 份 `allowed=True`。
- `对点练案1_已清洗.docx`：`status=approved`，高风险数为 0。

说明：直接收集全目录测试时，两个历史调试脚本会在导入阶段重包并关闭 `sys.stdout`，导致 pytest 收尾失败；本次改用根目录跨项目相关回归集完成验证，未修改该范围外的历史调试脚本。

## 结论

- 当前 44 份众美答案已通过自动检查，可直接进入 F2/F3/F4 录入。
- 后续重新清洗或合并阅读组时会自动刷新状态；人工改动 DOCX 后，文件签名变化仍会使旧状态失效。
