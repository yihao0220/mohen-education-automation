# 圆圈序号答案列点不再误触发F4

## 现象

- 答案正文使用 `①②③` 分点时，程序把三个要点识别成三个小问，连续触发三次 F4。
- 真实样本为对点练案41第9题：“①语言口语化……②连用动词……③句式灵活……”。
- 该题本身没有小问，正确行为应为整段答案按一次 F2，再按一次 F3 录入解析。

## 根因

- `shared_core/answer_core.py` 的答案项提取规则同时把 `(1)(2)` 和 `①②` 识别为小问，导致答案单元提前变成 `answer_mode=subquestion`。
- `答案录入/answer_input.py` 的执行层再次把 `①②` 当成小问边界，因此触发多次 F4。
- 44份众美已清洗答案中，共发现36个仅由圆圈序号造成的错误小问答案块。

## 改动点

- 共享答案解析层只把半角或全角括号数字 `(1)(2)…`、`（1）（2）…` 识别为小问。
- 交错“答案/解析”小问、段首小问和行内小问统一取消圆圈序号匹配。
- 答案录入执行层的 `find_subquestion_matches()` 只识别括号数字。
- `①②③…` 原样保留在单个 `AnswerItem` 内，整段使用 F2。
- 完整阅读组仍按题目分组生成 `(1)(2)…`，继续连续 F4；未来物理整段 F2 例外不变。

## 测试样例与结果

```powershell
python -m pytest -q test_zhongmei_chinese_answer_input.py -k "circled or lesson_41_question_9"
python -m pytest -q test_zhongmei_chinese_answer_input.py
python -m pytest -q test_zhongmei_chinese_answer_input.py test_zhongmei_chinese_workflow.py test_zhongmei_reading_answer_merge.py test_zhongmei_review_status.py test_zhongmei_chinese_answer_cleaner.py test_shared_core_flow.py test_zero_error_pipeline.py test_future_physics_workflow.py test_future_history_elective_two.py
python -m pytest -q test_future_physics_workflow.py -k "AnswerInput"
python -m pytest -q test_future_history_elective_two.py -k "circled_answer_points"
python -m py_compile shared_core\answer_core.py "答案录入\answer_input.py" test_zhongmei_chinese_answer_input.py
```

- 圆圈序号定向测试：3 passed。
- 众美答案录入专项：19 passed，108个 subtests 通过。
- 跨模块组合回归：本次相关91项通过，591个 subtests 通过；未来物理完整文件仍有历史已知的2个共享映射失败，本次未新增。
- 未来物理 F2 例外：2 passed。
- 未来历史圆圈列点保持单答案框：1 passed。
- 对点练案41第9题模拟按键：`F2 → F3`；F2完整包含 `①②③`，F2/F3均未混入“答案：”。
- 44份众美答案中仅由圆圈序号造成的小问块：修复前36个，修复后0个。

## 结论

- `①②③…` 统一作为一道答案内部的列点，不再触发 F4。
- 只有明确的括号数字小问或完整阅读分组才进入 F4 流程。
