# 众美高三物理解答题整组F2录入兼容

## 现象

物理解答题答案中可能出现 `(1)(2)`，但墨痕插件实际只保留一个答案框。若按通用小问规则触发多次 F4，会把答案录错位置；若题号写成 `1．(1)`，解析器还可能把它识别为复合题号并降级成顺序映射。

## 根因

- 小问文本结构与插件答案框结构并不总是一致。
- 未来高二物理已有“整组 F2”规则，但众美高三物理路径未纳入同一上下文。
- 强制整组录入场景下，`1．(1)` 需要用顶层题号 `1` 参与题答映射；只有 `(1)(2)` 占位、实际内容全在解析时，还应标记为“解析承载”，而不是低置信度错误。

## 改动点

- `答案录入/answer_input.py` 将 `众美-高三-物理` 纳入整组答案输入上下文，小问文本走一次 F2，不拆成多个 F4。
- `shared_core/answer_core.py` 为众美和未来两个物理项目统一设置 `force_whole_answer_input`。
- 强制整组场景下把 `1．(1)` 映射回顶层题号 `1`，并识别纯 `(2)` 等占位答案，让解析承担小问内容。
- 保留审核提示，但不再产生“题答顺序映射”或“答案块置信度较低”的误拦截。

## 测试样例

- 众美路径下 `answer_mode=subquestion`，`should_split_subquestion_answers()` 返回 `False`。
- 未来物理 `1．(2)CD (3)标记位置` 作为一道整题答案审核通过。
- 未来物理答案只含 `(1)(2)` 占位、解析含完整小问时，审核只给“解析承载”提醒，不产生高风险错误。

## 测试命令

```powershell
python -m unittest test_future_physics_workflow.FuturePhysicsSharedCoreTests.test_review_allows_future_physics_whole_answer_for_subquestions test_future_physics_workflow.FuturePhysicsSharedCoreTests.test_review_allows_future_physics_analysis_only_subanswers
python -m unittest test_zhongmei_physics_workflow.py test_future_physics_workflow.py test_shared_core_flow.py test_subject_detection.py test_zhongmei_review_status.py
```

结果：定向 2 条通过；相关回归合计 25 条通过。

## 结论

众美高三物理解答题沿用已验证的物理整组 F2 规则，复合题号和解析承载场景不会再被错误降级；未在测试中自动发送真实按键，避免对用户 WPS 插件产生外部录入动作。

## 下次排查入口

1. 按键选择先看 `答案录入/answer_input.py` 的 `should_split_subquestion_answers()`。
2. 题答映射先看 `shared_core/answer_core.py` 的 `_answer_mapping_question_id()` 和 `map_answers()`。
3. 若插件侧答案框结构改变，应以实际题型框为准重新决定 F2/F4，不要只看答案文本有没有 `(1)(2)`。
